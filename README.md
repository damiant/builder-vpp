# Fusion Metrics

A web application for tracking and visualizing metrics from the Fusion API. Monitor user activity, design exports, code merges, and other key metrics across your companies and spaces.

## Features

- **Multi-Company Support**: Manage and track metrics for multiple companies
- **API Integration**: Connect to the Fusion Metrics API using secure API credentials
- **Date Range Filtering**: View metrics for any month and year
- **Space Filtering**: Filter metrics by individual spaces or view all spaces at once
- **Metrics Dashboard**: Display key metrics including:
  - User prompts
  - Design exports
  - PRs merged
  - Lines of code
  - Credits used
  - User count
  - Events
- **Event Data Analysis**: Detailed breakdowns of event data by:
  - **Models**: View which AI models are being used, their performance, and credit consumption
  - **Projects**: Track metrics across different projects
  - **Features**: Analyze feature usage patterns
  - **Design vs Prompt**: Compare metrics between design exports and prompts, including event counts and credit usage
  - **Users**: Per-user metrics with model-level breakdowns
- **Data Export**: Multiple export options for sharing and analysis:
  - **CSV Export**: Export detailed event data with timestamps, user emails, spaces, projects, credits, and more
  - **PNG Export**: Export metrics dashboard as an image
  - **PDF Export**: Export complete metrics report with charts and tables
  - Filenames include company name and date range for easy organization
- **Data Visualization**: Charts and tables for easy metrics analysis
- **Local Caching**: Efficient caching of metrics and events data for improved performance
- **Responsive Design**: Clean, modern UI built with Lit and TailwindCSS

## Getting Started

### Prerequisites

- Node.js 18+ with npm
- Fusion API credentials (public and private keys)

### Installation

```bash
npm install
```

### Development

```bash
npm run dev
```

The application will start at `http://localhost:5173` (or your configured dev port).

### Build

```bash
npm run build
```

### Type Checking & Linting

```bash
npm run check
```

## Configuration

### Adding a Company

1. Click the **Add** button in the header
2. Enter the company name
3. Enter your Fusion Metrics API private key
4. Click **Connect** to verify the credentials
5. Once connected, click **Save**

### Editing Company Settings

1. Select a company from the dropdown
2. Click the **Edit** button
3. Update the company name or private key
4. Click **Save** to apply changes
5. Click **Delete** to remove the company permanently

## Data Export

### CSV Export

Export event data to CSV for use in external tools and analysis:

1. Click the **CSV** button in the header toolbar
2. The CSV file will download automatically with filename format: `{CompanyName}-{Month}-{Year}.csv`
3. CSV includes the following columns:
   - **timestamp**: Event timestamp
   - **userEmail**: Email of the user who triggered the event
   - **spaceName**: Name of the space
   - **spaceId**: ID of the space
   - **projectName**: Name of the project
   - **designExportId**: ID of the design export (if applicable)
   - **creditsUsed**: Credits consumed by the event
   - **linesOfCode**: Lines of code generated
   - **feature**: Feature used
   - **model**: AI model used

## Metrics Analysis

### Models Table

Displays aggregated metrics for each AI model used during the selected period:

- **Model**: Name of the AI model
- **Total Lines**: Total lines of code generated by this model
- **Events**: Number of times this model was used
- **Credits Used**: Total credits consumed by this model
- **Credits Per Event**: Average credits per event
- **Credit Distribution**: Visual bar showing relative credit consumption

### Design vs Prompt Analysis

Compare metrics between design exports and standard prompts:

- **Type**: Design or Prompt
- **Count**: Number of events in each category
- **Credits Used**: Total credits consumed
- **Unique Designs**: For designs, the count of unique design export IDs

### Projects, Features, and Users

Additional breakdowns available:

- **Projects**: Metrics aggregated by project name
- **Features**: Metrics aggregated by feature type
- **Users**: Per-user metrics including email and total credits used, with model-level breakdowns available

## API Integration

The application communicates with the Fusion Metrics API using bearer token authentication. The private key is sent in the `Authorization` header with the format:

```
Authorization: Bearer <private_key>
```

Two main API endpoints are used:

1. **Metrics Endpoint**: Fetches aggregated metrics for the selected date range
   - Returns: User prompts, design exports, PRs merged, lines of code, credits used, user count, events, space-specific metrics

2. **Events Endpoint**: Fetches detailed event records with pagination
   - Returns: Individual event data including user, model, feature, credits, and design export ID
   - Used for: Models, projects, features, and design vs prompt analysis

## Data Storage

- **Companies**: Stored in browser's IndexedDB for persistence across sessions
- **Metrics**: Cached locally to reduce API calls for the same date ranges
- **Events**: Cached after fetching to enable quick analysis without re-fetching

## Troubleshooting

### "Private key is required to fetch metrics"

The selected company doesn't have a valid API private key configured. Edit the company settings and enter a valid private key.

### Connection Successful but No Data

The Fusion Metrics API may not have data for the selected date range. Try selecting a different month/year or verify the company has active metrics.

### Metrics Not Loading

1. Check your internet connection
2. Verify the API credentials are correct
3. Check the browser console for detailed error messages
4. Try clearing the browser cache
